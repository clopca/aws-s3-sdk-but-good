Complete API reference for the `@s3-good/browser` package. This package provides a full-featured S3 file browser with React components, hooks, and a client-side API layer.

## Components

### S3Browser

The all-in-one file browser component. Renders a toolbar, file grid/list, search, breadcrumbs, dialogs, and preview modal.

```typescript
import { S3Browser } from "@s3-good/browser";
```

**Props:** `S3BrowserProps`

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `url` | `string` | `"/api/browser"` | API endpoint URL |
| `headers` | `HeadersInit \| (() => HeadersInit \| Promise<HeadersInit>)` | — | Custom request headers |
| `config` | `BrowserConfig` | — | Client-side configuration |
| `className` | `string` | — | CSS class for the container |
| `onNotify` | `(notification: S3BrowserNotification) => void` | — | Toast notification callback |
| `upload` | `object` | — | Upload configuration |
| `children` | `(ctx: S3BrowserRenderContext) => ReactNode` | — | Render prop for custom layouts |
| `pagination` | `S3BrowserPaginationOptions` | — | Pagination settings |
| `virtualization` | `{ grid?, list? }` | — | Virtualization options |
| `appearance` | `object` | — | CSS class overrides per slot |

See [S3 Browser docs](/components/s3-browser/) for full documentation.

### S3BrowserRoot

A headless context provider that runs `useBrowser` and exposes the result via React context. Use this to build custom browser UIs.

```typescript
import { S3BrowserRoot, useS3BrowserContext } from "@s3-good/browser";
```

**Props:** `S3BrowserRootProps`

| Prop | Type | Description |
|------|------|-------------|
| `url` | `string` | API endpoint URL |
| `headers` | `HeadersInit \| (() => HeadersInit \| Promise<HeadersInit>)` | Custom request headers |
| `config` | `BrowserConfig` | Client-side configuration |
| `children` | `ReactNode \| ((browser: UseBrowserReturn) => ReactNode)` | Children or render prop |

See [Headless Browser docs](/components/headless-browser/) for usage patterns.

## Hooks

### useBrowser

The core hook that manages all browser state — navigation, selection, sorting, search, and CRUD operations.

```typescript
import { useBrowser } from "@s3-good/browser";

function useBrowser(options?: UseBrowserOptions): UseBrowserReturn
```

**Options:** `UseBrowserOptions`

| Option | Type | Description |
|--------|------|-------------|
| `url` | `string` | API endpoint URL |
| `headers` | `HeadersInit \| (() => HeadersInit \| Promise<HeadersInit>)` | Custom request headers |
| `config` | `BrowserConfig` | Client-side configuration |

**Returns:** `UseBrowserReturn`

| Property | Type | Description |
|----------|------|-------------|
| `availableBuckets` | `string[]` | Buckets available for selection |
| `activeBucket` | `string` | Currently active bucket |
| `currentPath` | `string` | Current folder path |
| `items` | `BrowserItem[]` | Sorted items in the current folder |
| `selectedKeys` | `Set<string>` | Keys of selected items |
| `viewMode` | `ViewMode` | `"grid"` or `"list"` |
| `sort` | `SortConfig` | Current sort field and direction |
| `searchQuery` | `string` | Active search query |
| `filters` | `BrowserListFilters` | Active filters |
| `isLoading` | `boolean` | Whether a request is in flight |
| `error` | `string \| null` | Current error message |
| `hasMore` | `boolean` | Whether more pages are available |
| `previewItem` | `BrowserItem \| null` | Item being previewed |
| `navigateTo` | `(path: string) => void` | Navigate to a path |
| `openFolder` | `(item: BrowserItem) => void` | Open a folder item |
| `goBack` | `() => void` | Navigate back in history |
| `goForward` | `() => void` | Navigate forward in history |
| `goUp` | `() => void` | Navigate to parent folder |
| `select` | `(key: string) => void` | Select an item |
| `deselect` | `(key: string) => void` | Deselect an item |
| `toggleSelect` | `(key: string) => void` | Toggle item selection |
| `selectAll` | `() => void` | Select all items |
| `deselectAll` | `() => void` | Clear selection |
| `selectRange` | `(fromKey: string, toKey: string) => void` | Select a range |
| `setViewMode` | `(mode: ViewMode) => void` | Change view mode |
| `setSort` | `(field: SortField, direction?: SortDirection) => void` | Change sort |
| `setSearchQuery` | `(query: string) => void` | Set search query |
| `setFilters` | `(filters: BrowserListFilters) => void` | Set filters |
| `setActiveBucket` | `(bucket: string) => void` | Switch bucket |
| `refresh` | `() => Promise<void>` | Reload current folder |
| `loadMore` | `() => Promise<void>` | Load next page |
| `deleteSelected` | `() => Promise<void>` | Delete selected items |
| `renameItem` | `(key: string, newName: string) => Promise<void>` | Rename an item |
| `moveItem` | `(key: string, destination: string) => Promise<void>` | Move an item |
| `copyItem` | `(key: string, destination: string) => Promise<void>` | Copy an item |
| `createFolder` | `(name: string) => Promise<void>` | Create a folder |
| `downloadFile` | `(key: string) => Promise<void>` | Download a file |
| `setPreviewItem` | `(item: BrowserItem \| null) => void` | Set preview item |
| `store` | `BrowserStore` | Underlying store instance |
| `client` | `BrowserClient` | Underlying API client |

### useSearch

Debounced search hook that syncs an input value with the browser store's search query.

```typescript
import { useSearch } from "@s3-good/browser";

function useSearch(store: BrowserStore, debounceMs?: number): UseSearchReturn
```

**Returns:** `UseSearchReturn`

| Property | Type | Description |
|----------|------|-------------|
| `searchQuery` | `string` | Current committed search query |
| `inputValue` | `string` | Current input value (may differ during debounce) |
| `handleChange` | `(value: string) => void` | Input change handler |
| `clear` | `() => void` | Clear the search |
